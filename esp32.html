JPEG 硬解 TCP DMA 双核流水线

点击展开详情
项目描述

三任务流水线架构实现 MJPEG 视频流实时显示。TCP 服务端监听连接，接收任务解析帧协议获取 JPEG 数据，解码任务利用 ESP32-S3 硬件 JPEG 解码器转换为 RGB565，通过 SPI DMA 异步写入 LCD。双 RGB 缓冲区实现解码与 DMA 传输并行。

PSRAM 内存规划
  • LCD draw_buf 150KB 分配至 PSRAM，spi_bus_dma_memory_alloc 确保 DMA 对齐与 Cache 一致性
  • rgb_buf0 复用 LCD draw_buf，JPEG 解码输出零拷贝直送 SPI 传输
  • rgb_buf1 新增 150KB，heap_caps_aligned_alloc(16) + MALLOC_CAP_DMA 双缓冲交替
  • 4 个 JPEG 缓冲区共 128KB，显式指定 PSRAM 避免 32KB 分配落入内部 RAM
  • CONFIG_SPIRAM_MALLOC_ALWAYSINTERNAL=8192，小对象留内部 RAM 保实时性
  • WiFi/LwIP 缓冲区自动分配至 PSRAM，内部 RAM 预留 32KB 防挤占
网络层与帧协议
  • TCP_NODELAY 禁用 Nagle，SO_RCVBUF 扩至 32KB，应用层 16KB 滑动窗口
  • 帧协议：4 字节小端长度头 + JPEG 数据，SOI/EOI 双标记校验
  • 三级帧同步恢复：find_frame_sync（长度+SOI+EOI 三重验证）→ find_jpeg_soi → find_eoi_in_frame 截断恢复
  • 4 缓冲区计数信号量 + 原子索引无锁同步，Xtensa memw 内存屏障确保跨核可见
  • 解码慢于接收时清空信号量队列跳帧，只解码最新帧，宁可丢帧不延迟
  • LwIP 高速重传 + 乱序段排队，TCP 吞吐量测试含逐字节递增校验

项目描述

支持串口命令行与热点 Web 两种配网方式，覆盖开发调试与最终用户场景。Captive Portal 方案不依赖 ESP-IDF provisioning 框架，设备启动 SoftAP 后，DNS 服务器将所有域名劫持到 AP 地址，DHCP Option 114 通知客户端配网 URI，手机/电脑自动弹出配网页面。配网成功后凭据存入 NVS，后续上电自动连接。

核心特性
  • 串口命令行配网：join <SSID> <PASSWORD>，开发调试首选
  • 热点 Web 配网：prov start 启动 Captive Portal，最终用户无需串口
  • 自动配网策略：上电读取 NVS 凭据自动连接，失败则启动热点配网
  • SoftAP + HTTP Server + DNS 劫持，无需安装 APP
  • DHCP Option 114，手机自动弹出配网页面
  • 404 请求 303 重定向，确保所有 HTTP 请求跳转配网页
  • 中文配网界面，移动端适配，WiFi 扫描列表选择
  • NVS 凭据持久化，最多重试 5 次，配网成功后自动关闭热点
  • STA+AP 双模式，配网与通信可并行

项目描述

ESP32-S3 板级驱动层，包括 SPI LCD（ST7789 320x240 80MHz 像素时钟）、I2C IO 扩展器 XL9555（控制 LCD 电源/背光/复位、摄像头、按键）、PSRAM OCT 8 线 80MHz、16MB Flash 自定义分区表（双 OTA 3MB + FAT 存储 8.44MB）。串口命令行提供 6 个子命令模块，覆盖 LCD 测试、网络吞吐量测试、视频流控制等。

核心特性
  • ST7789 SPI LCD，80MHz 像素时钟，PSRAM DMA 全屏缓冲
  • XL9555 I2C 16 位 IO 扩展，管理 LCD/摄像头/按键
  • PSRAM OCT 8 线 80MHz，DMA 对齐分配
  • 16MB Flash 分区：factory 1.5MB + 双 OTA 3MB + FAT 8.44MB
  • ESP Console + argtable3，6 个命令模块
  • TCP/UDP 吞吐量测试，含丢包/乱序/重复检测
